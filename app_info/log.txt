🟡ログ
アプリケーションからのメッセージをファイル、システムエラーログ、Slackに記録して通知できる。

内部的にLaravelはMonologライブラリを利用している。



🔵ログのフォーマット
[日時] 環境.ログレベル: メッセージ コンテキスト



🔵ログの設定
ガイド「設定」

設定はconfig/logging.phpにある。

🟢チャンネル
..チャンネルごとにログが送られる場所や、形式を設定できる。

メッセージをログに記録するときに、デフォルトでstackチャンネルを使用する。
stackチャンネルは、複数のログチャンネルを単一のチャンネルに集約するために使用

ー　チャンネル名
nameセクション
ガイド「チャンネル名の設定」

ー　ドライバ（ログ記録の方法と場所はドライバの種類によって変わる）
deiverセクション
ガイド「利用可能なチャンネルドライバ」、「チャンネルの事前設定」

ー　チャンネルのカスタマイズ
ガイド「monologチャンネルカスタマイズ」

ー　ログレベル
levelセクション
設定したログレベル以上のレベルのメッセージのみ、チャンネルに送信される。
ガイド「ログレベル」

ー　非推奨の警告をログに残す
ガイド「廃止ワーニングのログ」



🔵stackチャンネル
ガイド「ログスタックの構築」

メッセージをログに記録するとき、デフォルトではstackチャンネルを使用する。

stackチャンネル...複数のログチャンネルを単一のチャンネルに集約したもの。

例）以下の設定のとき、stackチャンネルが使われるときには、dailyとslackのチャンネルに送られる。
※実際にログに記録するかどうかは、メッセージの重大度／「レベル」によって決定される。

'channels' => [
    'stack' => [
        'driver' => 'stack',
        'channels' => ['daily', 'slack'],
    ],

    'daily' => [
        ///
    ],

    'slack' => [
        ///
    ],
],
