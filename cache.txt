🟡キャッシュ

ー　キャッシュの役割
アプリケーションでデータ取得や処理に時間がかかる場合、結果を一時的に保存（キャッシュ）して、次回以降のリクエストで高速にデータを取得できるようにする。

ー　キャッシュの保存先
MemcachedやRedisなどの高速なデータストアを使用する。

ー　Laravelのキャッシュ機能
Laravelはキャッシュ操作のための統一されたAPIを提供しており、簡単にキャッシュを利用してアプリケーションを高速化することができる。



🔵設定
設定ファイル：config/cache.php

サポートされるキャッシュバックエンド：Memcached、Redis、DynamoDB、データベース、ファイルベースなど。
デフォルト設定: Laravelはデフォルトでdatabaseキャッシュを使用している。



🔵キャッシュの使用
Cacheファサードを使用する。

※詳細はガイド「キャッシュ使用法」



🔵ロック管理（競合の回避）
アトミックロックを使用すると、競合状態を気にすることなく分散ロックを操作できる。

$lock = Cache::lock('key', 10); // 10秒間ロック
if ($lock->get()) {
    // 処理
    $lock->release(); // ロック解除
}



🔵カスタムキャッシュドライバ
独自のキャッシュドライバを作成できる。



🔵キャッシュのイベント
すべてのキャッシュ操作でイベントが発火し、イベントをリッスンできる。
